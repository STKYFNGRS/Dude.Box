1. RESPONSIVE LAYOUT STANDARDS
----------------------------
Hero Section Standards:
- Mobile padding top: pt-40
- Desktop padding top: sm:pt-80
- Mobile padding bottom: pb-24
- Desktop padding bottom: sm:pb-48
- Maintain proper container structure:
```tsx
<div className="w-full flex justify-center">
  <div className="container px-4 sm:px-6 mx-auto">
    <div className="pt-40 sm:pt-80 pb-24 sm:pb-48 flex items-center justify-center">
      {/* Content */}
    </div>
  </div>
</div>
```

Content Section Standards:
- Use container class for consistent margins
- Implement responsive padding (px-4 sm:px-6)
- Maintain max-width constraints for readability
- Use responsive padding for different screen sizes

2. TYPE MANAGEMENT
-----------------
Best Practices:
- Define types in dedicated files under src/types/
- Avoid duplicate type definitions
- Export types that will be used across multiple files
- Use descriptive names for interfaces and types
- Document required fields with comments
- Use TypeScript strict mode
- Remove unused type imports to prevent ESLint errors
